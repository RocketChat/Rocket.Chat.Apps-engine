Class AppSlashCommandManager

The command manager for the Apps.

An App will add commands during their initialize method. Then once an App's onEnable is called and it returns true, only then will that App's commands be enabled.

Registered means the command has been provided to the bridged system.

addCommand

  • addCommand(appId: string, command: ISlashCommand): void
    • Defined in github_repo/RocketChatReposForked/Rocket.Chat.Apps-engine/src/server/managers/AppSlashCommandManager.ts:96

    Adds a command to be registered. This will not register it with the bridged system yet as this is only called on an App's initialize method and an App might not get enabled. When adding a command, it can not already exist in the system (to overwrite) and another App can not have already touched or provided it. Apps are on a first come first serve basis for providing and modifying commands.

    Parameters

    • appId: string

      the app's id which the command belongs to

    • command: ISlashCommand

      the command to add to the system

    Returns void

canCommandBeTouchedBy

  • canCommandBeTouchedBy(appId: string, command: string): boolean
    • Defined in github_repo/RocketChatReposForked/Rocket.Chat.Apps-engine/src/server/managers/AppSlashCommandManager.ts:60

    Checks whether an App can touch a command or not. There are only two ways an App can touch a command:

    1. The command has yet to be touched
    2. The app has already touched the command

    When do we consider an App touching a command? Whenever it adds, modifies, or removes one that it didn't provide.

    Parameters

    • appId: string

      the app's id which to check for

    • command: string

      the command to check about

    Returns boolean

    whether or not the app can touch the command
